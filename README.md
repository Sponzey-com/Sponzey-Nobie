`Sponzey Nobie`는 설치 후 빠르게 설정하고 바로 사용할 수 있도록 설계된 오케스트레이션 중심 개인 AI입니다. 제품 안에서 실제로 사용자와 대화하고 작업을 수행하는 이름은 `Nobie`입니다.

이 프로젝트의 목표는 단순한 채팅 AI가 아니라, 사용자의 요청을 이해하고, 적절한 실행 경로를 선택하고, 실제 로컬 작업과 외부 연결까지 이어서 처리하는 개인 작업 허브를 만드는 것입니다.

## 제품 해석

Nobie는 다음과 같이 이해하면 됩니다.

- `개인 비서형 AI`: 사용자의 컴퓨터와 작업 환경을 기준으로 동작합니다.
- `오케스트레이션 우선 AI`: 답변만 하는 것이 아니라, 어떤 작업을 어떤 방식으로 수행할지 판단하고 실행 흐름을 조율합니다.
- `설치 후 바로 쓰는 도구`: 컴퓨터에 익숙하지 않은 사용자도 WebUI에서 순서대로 설정하고 사용할 수 있도록 계속 적용 중입니다.
- `확장 가능한 작업 플랫폼`: AI 연결, 외부 기능 연결 `(MCP)`, 작업 능력 확장 `(Skill)`, 대화 채널 `(Communication)`을 순차적으로 적용 중입니다.

## 누구를 위한 제품인가

Sponzey Nobie의 핵심 타겟은 다음과 같습니다.

- 컴퓨터 사용이 익숙하지 않은 사용자
- AI, MCP, Skill, 채널 연동 같은 용어가 낯선 사용자
- 복잡한 개발자 설정 없이 바로 쓸 수 있는 개인 비서를 원하는 사용자

그래서 UI와 문서는 아래 원칙을 따릅니다.

- 쉬운 한국어를 먼저 보여줍니다.
- 원문 용어는 `()` 안에 보조 정보로 붙입니다.
- 필수 설정은 분명하게 표시합니다.
- 저장, 취소, 건너뛰기, 다음 같은 행동은 단순하고 확실해야 합니다.

## 핵심 기능

### 1. AI 연결 `(AI Backend)`

Nobie가 실제로 응답하고 작업을 수행하도록 AI를 연결합니다.

- OpenAI
- Claude
- Gemini
- Ollama
- OpenAI-compatible endpoint
- 로컬/원격 추론 서버

### 2. 외부 기능 연결 `(MCP)`

Nobie가 외부 도구와 기능을 확장해서 사용할 수 있도록 MCP 서버를 연결합니다.

- stdio 기반 MCP 서버 등록
- 도구 목록 조회
- 필수/선택 서버 구분
- 설정 화면에서 연결 상태 확인

### 3. 작업 능력 확장 `(Skill)`

특정 작업을 더 잘 수행하도록 작업 지침이나 능력 묶음을 붙입니다.

- 로컬 Skill 등록
- Skill 활성화/비활성화
- 설명과 출처 표시

### 4. 대화 채널 `(Communication)`

Nobie와 대화할 채널을 연결합니다.

- WebUI 채팅
- Telegram
- Slack 적용 예정

### 5. 로컬 작업 실행

Nobie는 로컬 컴퓨터에서 다음과 같은 작업을 수행할 수 있습니다.

- 파일 읽기/쓰기/검색/수정
- 셸 명령 실행
- 앱 실행
- 화면 캡처
- 마우스/키보드 제어
- 작업 큐, 승인, 취소, 검증 흐름 관리

## 설정 경험의 방향

Sponzey Nobie의 설정 화면은 `1920 x 1080` 기준으로 계속 정리하고 있는 적용 예정/적용 중 설계입니다.

기본 원칙은 다음과 같습니다.

- 한 단계는 한 화면 안에서 이해 가능해야 합니다.
- 긴 스크롤에 의존하지 않습니다.
- 고급 옵션은 접기 영역으로 분리합니다.
- 좌측은 단계 목록, 중앙은 현재 입력, 우측 또는 하단은 도움말과 상태를 보여줍니다.
- 하단 액션 영역은 항상 명확하게 보이도록 유지합니다.

설정 순서는 다음과 같습니다.

1. 개인 정보 입력
2. AI 연동
3. 외부 기능 연결 `(MCP)`
4. 작업 능력 확장 `(Skill)`
5. 대화 채널 `(Communication)`
6. 검토 및 완료

## 내부 동작 구조

Sponzey Nobie는 하나의 요청을 단순한 메시지로만 처리하지 않습니다.

현재 적용 중인 구조의 핵심 방향은 다음과 같습니다.

- `A = 검토 AI`
- `B-n = 작업 AI 또는 작업 세션`

역할 분리는 다음과 같습니다.

- `A`는 요청을 읽고, 무엇을 해야 하는지 정리하고, 어떤 실행 대상을 선택할지 판단하도록 적용 중입니다.
- `B-n`은 실제 파일 생성, 실행, 검증, 도구 사용을 담당하는 방향으로 적용 중입니다.
- 사용자가 보는 작업 큐는 `요청 하나 = 카드 하나`를 기준으로 유지하고, 그 안에 하위 작업이 트리처럼 붙도록 계속 정리 중입니다.

이 구조의 목적은 다음과 같습니다.

- 같은 채팅창에서 여러 요청이 들어와도 내용이 섞이지 않게 하기
- 승인, 취소, 재질의, 검증 상태를 요청 단위로 추적하기
- 루트 요청과 하위 작업을 명확히 구분해서 보여주기

## 프로젝트 구성

이 저장소는 대체로 세 부분으로 구성됩니다.

- `packages/core`
  - 에이전트 실행, 라우팅, 작업 큐, 검증, 설정 로직, 채널 연동, MCP, 도구 시스템
- `packages/cli`
  - daemon 실행과 로컬 명령 진입점
- `packages/webui`
  - 설정 UI, 채팅 UI, 작업 큐, 승인/취소/검토 화면

## 빠른 시작

### 요구 사항

- Node.js `22+`
- `pnpm`

### 설치

```bash
pnpm install
```

### 빌드

```bash
pnpm build
```

### 로컬 실행

```bash
bash scripts/start-local.sh
```

실행 후 기본 주소:

- Gateway: `http://127.0.0.1:18888`
- WebUI: `http://127.0.0.1:5173`

중지:

```bash
bash scripts/stop-local.sh
```

## 상태 디렉터리와 설정 파일

기본 상태 디렉터리는 다음 우선순위를 따릅니다.

- `NOBIE_STATE_DIR`
- 기본값 `~/.nobie`

여기에 다음 정보가 저장됩니다.

- 설정 파일
- setup 상태
- 로컬 DB
- 인증 토큰과 실행 상태

## 현재 범위와 주의 사항

현재 코드 기준으로 이해해야 할 점은 다음과 같습니다.

- WebUI 기반 설정과 작업 큐 구조가 중심으로 적용되어 있습니다.
- MCP client는 stdio 기반 서버 연결까지 적용되어 있고, 운영 경험은 계속 보완 중입니다.
- Telegram 채널은 연동되어 있습니다.
- Slack은 아직 적용 예정입니다.
- 일부 로컬 UI 자동화와 운영 흐름은 macOS 기준으로 더 많이 검증되어 있고, 다른 환경 지원은 적용 예정이 남아 있습니다.
- Windows는 일부 코드 경로만 있고, 정식 설치 지원은 아직 적용 예정입니다.

즉, Sponzey Nobie는 이미 동작하는 기능이 많지만, `작업 분리 아키텍처`, `검증`, `재시작 복구`, `운영 경험`은 계속 적용 중인 프로젝트입니다.

## 한 줄 정의

`Sponzey Nobie`는 사용자의 컴퓨터 위에서 동작하며, 요청을 이해하고, 적절한 실행 대상을 선택하고, 실제 작업과 검증까지 이어서 처리하는 오케스트레이션 중심 개인 AI 플랫폼입니다.
